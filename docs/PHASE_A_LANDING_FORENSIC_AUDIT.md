## E. Viewport limitation acknowledged

The Chrome Extension MCP on owner's laptop has a hard viewport floor — `resize_window` succeeds (returns "Successfully resized") but the Chrome content area cannot shrink below ~1620px on this display, so 1024 / 768 / 375 viewport screenshots could not be captured side-by-side.

**Compensating evidence:**
- DOM-level identity (section D.2 + D.3) is viewport-independent — same elements render at every breakpoint
- Both pages link to identical CSS (home-v2.css auto-prefixed from template's styles.css) — same media queries fire at the same breakpoints
- Owner can manually verify mobile/tablet via real device + browser DevTools Device Mode (D13 smoke)

This limitation is the same one R7.1+R7.2 documented when capturing Lighthouse mobile vs desktop — Chrome on Windows + extension MCP cannot enforce sub-1024 viewports.
